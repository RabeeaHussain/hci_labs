Week 1: Human Memory & Human–Machine Interaction

Human Memory

Our everyday activities rely heavily on memory. Memory stores factual knowledge, procedural knowledge, language, and sensory information. It also gives us our sense of identity by preserving past experiences.

Memory is the second part of the human information-processing model. There are three types of memory:

  • Sensory Memory: Buffers information from the senses (iconic, echoic, and haptic memory).
  • Short-Term Memory: Temporary storage for quick access; decays rapidly.
  • Long-Term Memory: Stores information for extended periods.

Sensory Memory

Sensory memory acts as a buffer for stimuli received through the senses. These memories are constantly overwritten by new information.

Short-Term Memory

Short-term memory acts as a scratch-pad for temporary recall. It can be accessed rapidly but information only lasts briefly.

Human–Machine Interaction

  • Agents respond according to their environment.
  • Memorization capacity affects interaction design.
  • Human capabilities must be mapped carefully to machine capabilities.
  • Difference between UI and UX is introduced.

Reference: Alan Dix et al., Human–Computer Interaction, 3rd Edition, Chapter 1, pages 27–28.

Week 2: Interaction, Laws & UI/UX

Communication Mode / Interaction / Language

Interaction between humans and computers depends on the language of the system and the language of the user.

Norman’s Theory of Interaction

Norman’s model divides interaction into execution and evaluation phases. The seven stages are:

  1. Establishing the goal
  2. Forming the intention
  3. Specifying the action sequence
  4. Executing the action
  5. Perceiving the system state
  6. Interpreting the system state
  7. Evaluating the system state

The model explains usability problems using:

  • Gulf of Execution: Difference between intended actions and system actions.
  • Gulf of Evaluation: Difficulty in interpreting system feedback.

Miller’s Law

Humans can hold approximately 7 ± 2 items in short-term memory. Chunking helps increase effective memory capacity.

Hick-Hyman Law

Decision time increases with the number of choices available. Users eliminate options by categorizing them rather than processing all choices linearly.

UI vs UX

UI is a subset of UX. UI focuses on visual and interactive elements, while UX focuses on the overall user experience.

UI Design Considerations

  • Page layout
  • Color scheme and font selection
  • Interactive elements
  • Wireframes and prototypes

Week 3: Interaction Languages & Execution

Core Language

Core language refers to the language of the system.It includes commands, system functions, icons, menus, and interface elements that the system understands and provides to the user.

This language is often technical and based on how the system is implemented. Designers must ensure that the core language is understandable and usable.

Task Language

Task language is the language used by the user to express goals. It represents the way users think about their goals and tasks in real-world terms.

Users usually express tasks in simple and natural language, without considering how the system internally works.

Gulf of Execution

The gulf of execution represents the mismatch between what the user wants to do and what the system allows them to do.

If users cannot easily figure out how to perform an action, the gulf of execution is large. Good interface design aims to minimize this gulf by making actions visible and intuitive.

Gulf of Evaluation

The gulf of evaluation is the gap between the system’s output and the user’s ability to understand whether their goal has been achieved.

Clear feedback, meaningful messages, and visible system states help reduce the gulf of evaluation.

Norman’s Theory

Norman’s theory helps designers identify where interaction breaks down and how to improve usability.

Week 4: Design Principles, Prototyping & Systems

Why Some Symbols Become Standards

Some symbols become standards because they are repeatedly used across systems and applications. Over time, users learn their meanings, which reduces cognitive load.

Examples include the floppy disk icon for saving and the magnifying glass for search. Standard symbols improve usability by increasing recognition rather than recall.

Design Rules

Design rules are general principles that guide interface design. These rules help ensure consistency, usability, and accessibility.

  • Consistency in layout and behavior
  • Visibility of system status
  • Feedback after user actions
  • Error prevention and recovery

Prototyping

Prototyping is the process of creating early models of a system to test ideas and design decisions before full development.

Prototypes can be low-fidelity (paper sketches) or high-fidelity (interactive digital models).

System

A system is a combination of hardware, software, users, and procedures working together to achieve a goal.

In HCI, system design focuses on how users interact with technology efficiently and safely.

System Development

System development involves several stages including analysis, design, implementation, and evaluation.

User-centered design ensures that users are involved throughout the development process.

Case Study Review

The “Who Paid” problem and the automatic syringe are examples used to identify usability issues in real-world systems.

These case studies highlight problems related to feedback, visibility, and user error, reinforcing the importance of good design.
