Norman’s Action Cycle

Seven Stages of Action

  1. Forming the goal
  2. Forming the intention
  3. Specifying the action
  4. Executing the action
  5. Perceiving the system state
  6. Interpreting the system state
  7. Evaluating the outcome

Gulfs of Execution and Evaluation

  • Gulf of Execution: Difficulty in translating goals into actions supported by the system.
  • Gulf of Evaluation: Difficulty in understanding system feedback and outcomes.

Application: Automatic Syringe

In the automatic syringe, the gulf of execution increases when dosage controls are unclear. The gulf of evaluation appears when feedback does not clearly confirm the delivered dosage.

GOMS Model

Core Components

  • Goals: What the user wants to achieve
  • Operators: Actions available to the user
  • Methods: Sequences of operators
  • Selection Rules: Criteria for choosing methods

Application: Toggle Switch

For a toggle switch, the goal is to change system state. Operators include pointing and clicking. The method is short and consistent, reducing cognitive load.

Comparison with Norman

While Norman’s model focuses on cognitive stages and feedback, GOMS emphasizes task structure and efficiency.

Keystroke-Level Model (KLM)

KLM Operators

  • K: Keystroke or button press
  • P: Pointing
  • H: Homing
  • M: Mental preparation
  • R: System response time

Execution-Time Prediction

KLM predicts task completion time by summing operator durations. This helps compare interaction efficiency objectively.

Application: Who Paid Problem

In the “Who Paid” problem, excessive mental (M) operators increase task time, highlighting inefficiency compared to simpler interfaces.
